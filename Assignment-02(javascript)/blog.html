Discuss the scope of var, let, and const

1. VAR:

Function Scope:

The variables which declared with var keyword are mainly function-scoped.We are only access them within the function where they are defined.

Hoisting:

Which variable are declared with keyword var hoisted to be top of their containing function or global scope.We can use them before they are declared but they are initially undefined until their actual declaration happend.

2. Let :

Block Scope:

Which variable are declared with keyword let are block-scoped.that's meaned they are accessible only within the block where they are mainly defined.A block can be a function,loop etc.

Hoisting:

Unlike var, variables declared with let are not hoisted to the top of their containing block. They are only accessible after the point where they are declared.

3. CONST :

Block Scope:

Similar with let and const provides block-level scope and also making it suitable for defining variables with a limited scope.

Immutability:

What sets const apart from var and let is its immutability after initialization. Once you assign a value to a const variable, you cannot reassign it.

Immutability sets const apart from var and let after initialization. Once we assign a value to a const variable, we cannot reassign it.It will be fixed.

Tell us the use cases of null and undefined

null:

"null" is often used for an intentional absence of any object value or to explicitly indicate the absence of a value.

1. Initializing Variables:

We can assign null variables if we want to indicate that they haven't been assigned a valid value yet.

2 .Resetting Values:

"null" can used to reset value of a variable or property to a empty state or to indicate - it is no longer considered valid.

3. Checking for Absence:

We use null to explicitly check if a variable or property missed or haven't set yet.

undefined:

"undefined" is a value which indicates a variable or property has been declared but haven't assigned any value or it can be "null".

1. Default Value:

When a variable is declared, not assigned a value then it automatically assigned the value undefined.

2. Function Parameters:

If a function called with fewer arguments than declared parameters and the missing parameters are set to undefined.

3. Object Properties:

When we try to access an object property which isn't exist then the result is "undefined".

What do you mean by REST API?

What is a REST API?

REST is mainly "Representational State Transfer". It is an architectural style for designing networked applications and defines a set of constraints and principle to create web services.

Resources: In REST, everything is considered a resource in a REST API.A resource can be a data ,or an object or services. Each resource is identified by a unique URL.

HTTP Methods: REST APIs use standard HTTP methods like GET, POST, PUT, DELETE, and PATCH to perform actions on resources. For example, GET is used to retrieve data, while POST is used to create new resources.

Stateless: REST is stateless means that each request from client to a server must contain all the information which needed to understand & process the request.

Uniform Interface: REST APIs have an uniform and consistent interface to work with them that makes it easier for developers by using standard HTTP status codes for responses.
